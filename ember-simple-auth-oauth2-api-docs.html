Ember.SimpleAuth.Authenticators.OAuth2

Authenticator that conforms to OAuth 2 (RFC 6749), specifically the "Resource Owner Password Credentials Grant Type".

This authenticator supports access token refresh (see RFC 6740, section 6).

The factory for this authenticator is registered as 'ember-simple-auth-authenticator:oauth2-password-grant' in Ember's container.

Properties

source

serverTokenEndpoint String

The endpoint on the server the authenticator acquires the access token from.

Defaults to '/token'
source

refreshAccessTokens Boolean

Sets whether the authenticator automatically refreshes access tokens.

Defaults to true

Methods

source

restore(data) → Ember.RSVP.Promise

Restores the session from a set of session properties; will return a resolving promise when there's a non-empty access_token in the data and a rejecting promise otherwise.

This method also schedules automatic token refreshing when there are values for refresh_token and expires_in in the data and automatic token refreshing is not disabled (see Ember.SimpleAuth.Authenticators.OAuth2#refreshAccessTokens).

Arguments

data Object

The data to restore the session from

Returns

→ Ember.RSVP.Promise

A promise that when it resolves results in the session being authenticated

source

authenticate(credentials) → Ember.RSVP.Promise

Authenticates the session with the specified credentials; the credentials are POSTed to the serverTokenEndpoint (see Ember.SimpleAuth.Authenticators.OAuth2#serverTokenEndpoint) and if they are valid the server returns an access token in response (see http://tools.ietf.org/html/rfc6749#section-4.3). If the credentials are valid and authentication succeeds, a promise that resolves with the server's response is returned, otherwise a promise that rejects with the error is returned.

This method also schedules automatic token refreshing when there are values for refresh_token and expires_in in the server response and automatic token refreshing is not disabled (see Ember.SimpleAuth.Authenticators.OAuth2#refreshAccessTokens).

Arguments

credentials Object

The credentials to authenticate the session with

Returns

→ Ember.RSVP.Promise

A promise that resolves when an access token is successfully acquired from the server and rejects otherwise

source

invalidate() → Ember.RSVP.Promise

Cancels any outstanding automatic token refreshes and returns a resolving promise.

Returns

→ Ember.RSVP.Promise

A resolving promise

source

makeRequest(data) → Deferred object

Sends an AJAX request to the serverTokenEndpoint. This will always be a "POST" request with content type "application/x-www-form-urlencoded" as specified in RFC 6749.

This method is not meant to be used directly but serves as an extension point to e.g. add "Client Credentials" (see RFC 6749, section 2.3).

Arguments

data Object

The data to send with the request, e.g. username and password or the refresh token

Returns

→ Deferred object

A Deferred object (see the jQuery docs) that is compatible to Ember.RSVP.Promise; will resolve if the request succeeds, reject otherwise

Events

Triggered when the authenticator refreshes the access token (see RFC 6740, section 6).

Parameters

The updated session data

Ember.SimpleAuth.Authorizers.OAuth2

Authorizer that conforms to OAuth 2 (RFC 6749) by sending a bearer token (RFC 6749) in the request's Authorization header.

The factory for this authorizer is registered as 'ember-simple-auth-authorizer:oauth2-bearer' in Ember's container.

Methods

source

authorize(jqXHR, requestOptions)

Authorizes an XHR request by sending the access_token property from the session as a bearer token in the Authorization header:

Authorization: Bearer <access_token>

Arguments

jqXHR JqXHR

The XHR request to authorize (see http://api.jquery.com/jQuery.ajax/#jqXHR)

requestOptions Object

The options as provided to the $.ajax method (see http://api.jquery.com/jQuery.ajaxPrefilter/)
